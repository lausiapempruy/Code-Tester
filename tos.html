Pasal 01

Penerimaan Ketentuan

Dengan mengakses, mendaftar, atau menggunakan VeaCode 2.0 ("Layanan"), kamu ("Pengguna") menyatakan telah membaca, memahami, dan menyetujui untuk terikat oleh Ketentuan Layanan ini ("ToS").

Jika kamu tidak menyetujui ketentuan ini, kamu tidak diizinkan untuk membuat akun VeaCode 2.0. Penggunaan editor tanpa akun tetap diperbolehkan dan tidak memerlukan persetujuan ToS secara eksplisit.

Dengan mencentang "Saya menyetujui Terms of Service VeaCode 2.0" saat pendaftaran, kamu secara elektronik menyetujui seluruh ketentuan yang tercantum dalam dokumen ini.

Pasal 02

Deskripsi Layanan

VeaCode 2.0 adalah sebuah code playground berbasis browser yang berjalan sepenuhnya secara lokal di perangkat pengguna. Layanan ini menyediakan:

  • Editor kode multi-file yang mendukung HTML, CSS, JavaScript, JSON, dan Python (preview statis)
  • Preview real-time melalui iframe sandbox
  • Panel console untuk menangkap output JavaScript
  • Sistem akun lokal berbasis localStorage dengan autentikasi PIN
  • Dashboard pengguna, Settings akun, dan dokumentasi terintegrasi

VeaCode 2.0 tidak memiliki backend server, tidak menyimpan data di cloud, dan tidak menyediakan fitur kolaborasi real-time antar pengguna.

Pasal 03

Akun Pengguna

Akun VeaCode 2.0 bersifat device-bound — terikat pada satu perangkat dan browser tempat akun dibuat. Tidak ada mekanisme sinkronisasi lintas perangkat.

⚠ Akun VeaCode tidak bisa digunakan di beberapa perangkat secara bersamaan. Kami mohon maaf atas ketidaknyamanan ini.

Pengguna bertanggung jawab sepenuhnya atas:

  • Keamanan perangkat yang digunakan untuk mengakses akun
  • Menjaga kerahasiaan PIN akun
  • Seluruh aktivitas yang dilakukan melalui akun mereka
  • Membuat backup data secara mandiri melalui fitur Ekspor Data

VeaCode tidak bertanggung jawab atas kehilangan akses akibat lupa PIN, kerusakan perangkat, atau penghapusan data browser.

Pasal 04

Username & Display Name

Username adalah identitas unik akun di perangkat tersebut. Ketentuan username:

  • Hanya boleh mengandung huruf (a–z, A–Z), angka (0–9), dan underscore (_)
  • Tidak boleh mengandung spasi, simbol khusus, atau karakter unicode
  • Minimal 3 karakter, maksimal 32 karakter
  • Username yang sudah digunakan tidak dapat digunakan ulang di perangkat yang sama
  • Penggantian username memerlukan verifikasi PIN

Display Name adalah nama yang ditampilkan di antarmuka. Ketentuan display name:

  • Bersifat opsional — jika tidak diisi, display name akan mengikuti username
  • Boleh berisi spasi dan karakter umum
  • Maksimal 64 karakter
  • Dapat diubah kapan saja tanpa verifikasi PIN

Pengguna dilarang menggunakan username atau display name yang:

  • Mengandung kata-kata kasar, melecehkan, atau menyinggung pihak lain
  • Menyamar sebagai tokoh publik, brand, atau entitas lain
  • Mengandung informasi pribadi orang lain tanpa izin

Pasal 05

PIN & Keamanan

PIN VeaCode 2.0 adalah kode numerik 10 digit yang berfungsi sebagai satu-satunya mekanisme autentikasi akun.

⚠ PIN tidak dapat dipulihkan jika hilang atau lupa. VeaCode tidak menyimpan PIN dalam bentuk apapun yang memungkinkan pemulihan. Simpan PIN kamu di tempat yang aman.

Ketentuan PIN:

  • PIN wajib terdiri dari tepat 10 digit angka
  • PIN disimpan dalam bentuk hash SHA-256 — PIN asli tidak pernah tersimpan dalam plaintext
  • PIN diperlukan untuk: login, ganti username, ganti PIN, dan verifikasi tertentu di Settings
  • Penggantian PIN memerlukan PIN lama sebagai verifikasi dan membutuhkan waktu sekitar 10 detik untuk diproses
  • Setelah ganti PIN, pengguna akan diminta login ulang

Pengguna dilarang:

  • Mencoba melakukan brute-force terhadap PIN akun sendiri atau orang lain
  • Memanipulasi localStorage secara langsung untuk membypass autentikasi PIN

Pasal 06

Proyek & Kode

Seluruh kode yang ditulis di editor VeaCode 2.0 adalah milik pengguna sepenuhnya. VeaCode tidak mengklaim hak apapun atas konten kode yang dibuat.

Pengguna bertanggung jawab penuh atas kode yang mereka tulis. Pengguna dilarang menggunakan editor VeaCode untuk:

  • Menulis atau menguji malware, virus, ransomware, atau kode berbahaya lainnya
  • Membuat konten yang melanggar hukum yang berlaku
  • Menguji teknik serangan siber terhadap sistem pihak ketiga
  • Menghasilkan konten yang menyebarkan kebencian, diskriminasi, atau kekerasan
  • Melanggar hak kekayaan intelektual pihak lain

Fitur Code Projects (penyimpanan proyek permanen) saat ini belum tersedia dan akan hadir di pembaruan berikutnya.

Pasal 07

Penyimpanan Data Lokal

VeaCode 2.0 menggunakan localStorage browser sebagai satu-satunya media penyimpanan. Ini berarti:

  • Data tersimpan hanya di browser spesifik pada perangkat yang digunakan
  • Membersihkan data browser (clear site data) akan menghapus seluruh data VeaCode termasuk akun
  • Menggunakan mode incognito/private tidak akan mempertahankan data setelah tab ditutup
  • Data tidak dapat diakses dari browser lain meskipun di perangkat yang sama

VeaCode sangat menyarankan pengguna untuk secara berkala melakukan ekspor data sebagai backup, terutama sebelum membersihkan data browser.

⚠ VeaCode tidak bertanggung jawab atas kehilangan data akibat penghapusan cache, uninstall browser, reset perangkat, atau sebab teknis lainnya.

Pasal 08

Privasi & Tidak Ada Backend

VeaCode 2.0 dirancang dengan prinsip privacy by design:

  • Tidak ada server backend — tidak ada data yang dikirim ke luar perangkat
  • Tidak ada telemetri, analytics, atau tracking penggunaan
  • Tidak ada cookie pihak ketiga
  • Tidak ada iklan
  • Data akun (username, display name, avatar, PIN hash) hanya tersimpan di localStorage perangkat pengguna

Satu-satunya koneksi eksternal yang dilakukan VeaCode adalah:

  • Memuat font dari Google Fonts API (bisa diblokir jika diperlukan)
  • Memuat library eksternal dari CDN yang kamu tambahkan sendiri di kode HTML
VeaCode tidak pernah melihat, membaca, atau mengakses kode yang kamu tulis. Semuanya berjalan murni di browser kamu.

Pasal 09

Ekspor & Impor Data

VeaCode menyediakan fitur Ekspor dan Impor data untuk memudahkan backup dan pemindahan data antar sesi.

Ekspor Data:

  • Menghasilkan file JSON berisi seluruh data akun (username, display name, avatar, aktivitas, PIN hash)
  • Proses membutuhkan 5–10 detik
  • File ekspor adalah tanggung jawab pengguna sepenuhnya — simpan di tempat aman
  • PIN tersimpan dalam bentuk hash dan tidak bisa digunakan untuk mengetahui PIN asli

Impor Data:

  • Memuat data dari file JSON yang sebelumnya diekspor dari VeaCode
  • Proses membutuhkan 20–30 detik untuk validasi dan penerapan
  • Data yang ada akan ditimpa — pastikan sudah melakukan backup sebelum impor
  • VeaCode tidak bertanggung jawab atas kerusakan data akibat impor file yang tidak valid atau rusak

Pasal 10

Penggunaan yang Dilarang

Pengguna dilarang menggunakan VeaCode 2.0 untuk:

  • Aktivitas ilegal berdasarkan hukum yang berlaku di yurisdiksi pengguna
  • Mengembangkan, menguji, atau mendistribusikan perangkat lunak berbahaya
  • Melakukan reverse engineering terhadap sistem VeaCode itu sendiri dengan tujuan merusak
  • Mengeksploitasi atau mencoba mengeksploitasi kerentanan keamanan
  • Memanipulasi localStorage secara langsung untuk membypass mekanisme keamanan
  • Membuat konten yang melanggar hak cipta atau kekayaan intelektual
  • Menggunakan layanan untuk tujuan yang merugikan pihak ketiga

Pelanggaran terhadap ketentuan ini sepenuhnya menjadi tanggung jawab pengguna.

Pasal 11

Keterbatasan Layanan

VeaCode 2.0 memiliki keterbatasan bawaan yang perlu dipahami pengguna:

  • Single device: Akun hanya bisa diakses dari satu perangkat dan browser
  • Tidak ada sinkronisasi: Tidak ada backup otomatis ke cloud
  • Python tidak dieksekusi: File .py hanya ditampilkan sebagai teks, tidak dijalankan
  • Tidak ada save permanen untuk kode: Fitur Code Projects belum tersedia di v2.0
  • Bergantung pada browser: Fitur backdrop-filter dan localStorage memerlukan browser modern
  • PIN tidak dapat dipulihkan: Tidak ada mekanisme reset PIN

Fitur-fitur berikut direncanakan untuk rilis berikutnya namun belum tersedia di v2.0:

  • Code Projects (simpan proyek secara permanen)
  • Template starter
  • Syntax highlighting penuh

Pasal 12

Penolakan Garansi

VeaCode 2.0 disediakan "sebagaimana adanya" (as-is) tanpa garansi apapun, baik tersurat maupun tersirat, termasuk namun tidak terbatas pada:

  • Garansi ketersediaan atau uptime layanan
  • Garansi keamanan data dari kehilangan akibat faktor browser atau perangkat
  • Garansi kebebasan dari bug atau error
  • Garansi kesesuaian untuk tujuan tertentu

VeaCode dan pengembangnya tidak bertanggung jawab atas kehilangan data, kerusakan perangkat, atau kerugian lain yang timbul dari penggunaan layanan ini, sejauh yang diizinkan oleh hukum yang berlaku.
